# AWS Prerequisites

Before diving into AWS networking, ensure you understand these foundational AWS concepts. These services and concepts are essential for implementing networking solutions effectively.

## Essential AWS Knowledge

### AWS Account Structure

* **AWS Accounts** - Understand account boundaries and resource isolation
* **AWS Organizations** - Multi-account management and governance
* **Account strategies** - When to use single vs. multiple accounts

### Identity and Access Management (IAM)

* **Users, Groups, and Roles** - Authentication and authorization basics
* **Policies** - Permission management and least privilege principles
* **Cross-account access** - Sharing resources between accounts securely

### Core Infrastructure Concepts

* **Regions and Availability Zones** - Geographic distribution and high availability
* **Resource naming** - Consistent tagging and naming conventions
* **Service limits** - Understanding quotas and how to request increases

* [AWS Organizations User Guide](https://docs.aws.amazon.com/organizations/latest/userguide/) - Multi-account management

## Knowledge Check

Before proceeding with networking topics, you should be comfortable with:

* [ ] Creating and managing AWS accounts
* [ ] Understanding IAM roles and policies
* [ ] Working with AWS Organizations (if using multiple accounts)
* [ ] Basic AWS CLI or Console navigation
* [ ] Understanding AWS service limits and quotas
* [ ] Applying consistent resource tagging strategies

## Next Steps

DIAGRAM: IPAM Pool Hierarchy (Dual-Stack)
PURPOSE: Shows the AWS IPAM pool hierarchy from the IPAM instance through scopes, top-level pools, regional pools, environment pools, down to individual VPC allocations. Each pool level has separate IPv4 and IPv6 pools, visualized as stacked layers within a single card-style node. Demonstrates how IPAM provides centralized dual-stack IP governance at scale.
LAYOUT: top-to-bottom (tree structure)
WIDTH: 700px maximum readable width

GROUPS:
- No explicit group containers — this is a tree diagram

NODE STYLE:
Each pool node is rendered as a "stacked card" — a single rectangular box divided horizontally by a thin line into two rows:
- Top row: IPv4 pool (blue-tinted background)
- Bottom row: IPv6 pool (purple-tinted background)
This visually communicates that IPv4 and IPv6 are separate pools within IPAM, but managed in parallel at the same hierarchical level.

NOTES:
- BACKGROUND: Transparent (no white background). All text and lines that appear directly on the transparent background must use #7E7E7E instead of black (#000000). Text and lines inside colored nodes/shapes can remain as-is (white on colored fill is fine).
- The "stacked card" style is the key visual: each node is ONE box with a horizontal divider, showing IPv4 (top, blue tint) and IPv6 (bottom, purple tint) as separate-but-parallel pools
- This accurately represents IPAM's actual structure: IPv4 and IPv6 are separate pools, but managed at the same hierarchical level
- Tree flows top-to-bottom, widening at each level
- IPv6 subnetting mirrors IPv4: /32 → /36 → /37 → /48 (parallel to /8 → /12 → /13 → /16)
- The stacked visual should be immediately understandable: "at every level, there are two pools — one for each address family"

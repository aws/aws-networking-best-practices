DIAGRAM: Service Monitoring Stack
PURPOSE: Shows the three-part service monitoring architecture: Networking Service Metrics (TGW, NAT, DX, VPN, ALB, NLB, NFW, Route 53, Lattice) feeding into a Centralized Monitoring Account (CloudWatch metrics, alarms, dashboards) which then triggers Automated Response (EventBridge, SNS, Lambda, Incident Manager).
LAYOUT: left-to-right (three-column flow)
WIDTH: 700px maximum readable width

- Centralized Monitoring Account: border style dashed, color purple (#7c3aed); contains CloudWatch components
- Automated Response: border style dashed, color green (#059669); contains response targets

# Introduction

AWS Networking Architecture guidance and best practices by the AWS user community, vetted by AWS Networking Specialist.

## Tenets

### Focus on Real-World Use Cases

* Center the content around practical use cases drawn from field experiences rather than individual services.
* Emphasize networking scenarios that are frequently encountered in the trenches, offering insights that resonate with the daily challenges of SAs and TAMs.

### Prioritize Strong Fundamentals

* Concentrate on fundamental networking principles that are critical for robust architecture and design.
* Avoid speculative or cutting-edge possibilities, instead reinforcing the core concepts that underpin successful network strategies.

### IPv6 as a Core Element

* Treat IPv6 as a primary consideration, integrating it into examples, diagrams, and discussions as the standard.
* Ensure that all networking best practices include IPv6 as a fundamental component, reflecting its importance in modern networks.

### Technical Precision and Clarity

* Maintain a high level of technical accuracy, using clear, concise language tailored for a knowledgeable audience.
* Provide links to existing validated content where appropriate, ensuring readers have access to deeper resources without duplicating existing information.

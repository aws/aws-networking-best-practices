DIAGRAM: Perimeter Control Layers
PURPOSE: Shows the layered defense-in-depth model for inbound perimeter controls, from Edge (global) through VPC Boundary, Subnet, down to Resource layer. Traffic flows inward through these layers sequentially, and each layer can independently deny traffic.
LAYOUT: top-to-bottom (sequential layers)
WIDTH: 700px maximum readable width

GROUPS:
- Edge Layer (Global): border style dashed, color orange (#ff9900); contains CloudFront+WAF, Shield, Global Accelerator
- VPC Boundary Layer: border style dashed, color blue (#2563eb); contains Network Firewall, GWLB, NACLs
- Subnet Layer: border style dashed, color purple (#7c3aed); contains Security Groups
- Resource Layer: border style dashed, color green (#059669); contains ENI

NOTES:
- BACKGROUND: Transparent (no white background). All text and lines that appear directly on the transparent background must use #7E7E7E instead of black (#000000). Text and lines inside colored nodes/shapes can remain as-is (white on colored fill is fine).
- Four layers stacked top-to-bottom showing traffic flowing inward
- Arrows between layers show sequential evaluation
- Orange for edge (global AWS), blue for VPC boundary, purple for subnet-level, green for resource-level
- Each layer can independently block traffic — this is defense in depth
- The visual should clearly show the narrowing funnel from internet to resource
